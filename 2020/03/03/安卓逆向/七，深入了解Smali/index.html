七，深入了解Smali

Smali中的包信息

  • .class public Lcom/aaaaa ; 它是com.aaaaa这个packge下的一个类
  • .super Lcom/bbbbb; 继承自com.bbbbb这个类
  • .source “ccccc.java” 它是一个由ccccc.java编译得到的smali文件

Smali中的声明

一般来说在Smali文件中是如下格式：

#annotations
.annotation system Ldalvik/annotation/MemberClasses;
value = {
Lcom/aaa$qqq;,
Lcom/aaa$www;
}
.end annnotation

这个声明式内部类的声明：aaa这个类有两个成员内部类qqq和www。memberclasses就是成员类的意思

关于寄存器的知识补充

​ 在smali里的所有操作都必须经过寄存器来进行：本地寄存器用v开头数字结尾的符号表示，如v0 参数寄存器则使用p开头数字结尾的符号来表示，如p0 特别注意的是，p0不一定式函数中的第一个参数，在非static函数中，p0代指this指针，p1表示函数的第一个参数，，而在static函数中p0才对应第一个参数（因为在java的static方法中没有this方法）

寄存器简单实例分析

​ const/4 v0,0x1

​ iput-boolean v0,p0,Lcom/aaa;->IsRegistered:Z

​ 对这两段smali代码进行分析：首先它使用了v0本地寄存器，并把值0x1存到v0中，然后第二局用iput-boolean这个指令把v0中的值存放到com.aaa.IsRegistered这个成员变量中。

​ 即相当于：this.IsRegistered=true;

smali中的成员变量

  • 成员变量格式：.field public/private [static] [final] varName:<类型>
  • 对于不同的成员变量也有不同的指令
  • 一般来说，获取的指令有：iget，sget，iget-boolean，sget-boolean,iget-object,sget-object等
  • 操作的指令有：iput，sput，iput-boolean,sput-boolean,iput-object,sput-object等（get与put的区别）
  • 带object操作的是对象类型

Smali成员变量指令解析

  • sget-object v0,Lcom/aaa;->ID:Ljava/lang/String;

  • sget-object就是用来获取变量值并保存到紧接着的参数的寄存器中，本例中，获取ID这个string类型的成员变量并放到v0这个寄存器中

  • 注意：前面需要该变量所属的类的类型，后面需要加一个冒号和该成员变量的类型，中间式->表示所属关系

  • iget-object v0 ,p0,Lcom/aaa;->view:Lcom/aaa/view;

  • 可以看到iget-object指令比sget-object多了一个参数，就是该变量所在类的实例，在这里就是p0即this

  • 获取数组array的话我们用aget和aget-object

  • put指令的使用和get指令的统一的如下：

    • const/4 v3,0x0
    • sput-object v3,Lcom/aaa;->timer:Lcom/aaa/timer;
    • 相当于：this.timer=null;
    • 这里因为是赋值object所以是null
  • .local v0,args:Landroid/os/Message;
const/4 v1,0x12
iput v1,v0,Landroid/os/Message;->what:l

  • 相当于：args.what=18;(args是Message的实例)
