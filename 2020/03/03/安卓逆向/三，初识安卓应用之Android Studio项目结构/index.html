初识安卓应用之Android Studio项目结构

结构概览图

![image-20200303134508543](三，初识安卓应用之Android Studio项目结构/image-20200303134508543.png)

从上往下看逐个分析了解：

1，mainfests

​ 用于存放AndroidManifest.xml文件。

2，AndroidManifest.xml

概述：

​ 又称清单文件，该文件是整个项目的配置文件。在程序中定义的四大组件都需要在这个文件中注册，包含了Android系统运行Android程序前所要掌握的重要信息，比如：应用程序名称、图标、包名称、模块组成、授权和sdk最低版本等，这个文件还可以给程序添加权限。在清单文件中配置的信息会添加到Android系统中，当程序运行时，系统会找到清单文件中的配置信息，然后根据配置信息打开相应组件。

逐步分析：

xml标签：
<?xml version="1.0 encoding="utf-8"?>

声明xml的版本信息和编码方式

manifest：
<manifest xmlns:android="http://shemas.android.com/apk/res/android" package="com.example.shinelon.myapplication">
    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:theme="@style/AppTheme" >
        <activity
            android:name=".MainActivity"
            android:label="@string/app_name" >
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />

                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
    </application>

</manifest>
xmlns:android：

定义了Android的命名空间

package：

定义了应用程序的包名称

application：

manifest元素仅能够包含一个application元素，application元素中能够声明Android程序中最重要的四个组成部分，包括Activity、Service、BroadcastReceiver和ContentProvider，所定义的属性将影响所有组成部分

android:allowBackup:

自己判断是否允许备份

android.icon:

定义了Android应用程序的图标，其中@mipmap/ic_lanuncher是一种资源引用方式，表示资源类型是图像，资源名称为icon，对应的资源文件为res/mipmap目录下的ic_lanuncher.png

android.label:

定义了Android应用程序的标签名称 ，在string.xml文件中。

3，java

用于存放所有的Java代码，在该文件夹中可以创建多个包，每个包中可以存放不同的文件或Activity

4，res

用于存放Android程序所用到的资源，如：图片布局文件、字符串等

drawable

该目录用于存放图片及xml文件

layout

该目录用于存放布局文件

mipmap

通常用于存放应用程序图标，系统会根据手机屏幕分辨率匹配响应大小的图标

values

用于放置定义的字符串

5，Gradle Scripts

用于存放项目创建的相关文件，无需修改
